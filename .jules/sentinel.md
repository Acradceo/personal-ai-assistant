## 2024-05-09 - Mass Assignment Vulnerabilities in API Endpoints\n**Vulnerability:** PUT endpoints allowed dictionary updates with the entire JSON payload via `update(data)` directly to the stored object, permitting users to overwrite restricted fields like `id` and `created_at` or inject arbitrary fields.\n**Learning:** The in-memory objects were implicitly trusting user input for the full entity payload, which bypasses role and integrity checks.\n**Prevention:** Use explicit parameter whitelisting (extracting only specific allowed keys such as `title`, `description`, `status`) to construct updates before mutating the entity.
